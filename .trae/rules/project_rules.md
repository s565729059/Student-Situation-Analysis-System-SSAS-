# 项目规则

## 版本管理规则

### 小版本更新（功能调整）
每次进行功能调整/优化/bug修复时，必须自动执行以下步骤：
1. **版本号 +0.0.1**：例如 v3.3.1 → v3.3.2
2. **更新时间**：同步更新所有HTML文件中的版本号（index.html、exam-analysis.html、student-analysis.html、student-exam-analysis.html）
3. **版本日志**：在 index.html 的更新日志区域顶部新增一个版本条目，包含：
   - 版本号标签
   - 当前日期
   - 更新内容列表（用新增/优化/修复等标签分类）

### 大版本更新
仅当用户明确说"大版本更新"或"x.x大版本"时才执行大版本号变更（如 v3.x → v4.0），否则默认只加0.0.1。
