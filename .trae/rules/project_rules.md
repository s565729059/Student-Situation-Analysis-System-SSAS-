# 项目规则

## 版本管理规则

### 小版本更新（功能调整）
每次进行功能调整/优化/bug修复时：
1. **本地修改阶段**：先在本地完成代码修改，不更新版本号和日志
2. **推送阶段**：当用户明确说"推送"时，执行以下操作：
   - **版本号 +0.0.1**：例如 v3.3.1 → v3.3.2
   - **同步版本号**：更新所有HTML文件中的版本号（index.html、exam-analysis.html、student-analysis.html、student-exam-analysis.html）
   - **更新日志**：在 index.html 的更新日志区域顶部新增一个版本条目，包含：版本号标签、当前日期、更新内容列表（用新增/优化/修复等标签分类）
   - **总结更新**：总结这段时间的所有更新情况
   - **执行推送**：git add + git commit + git push

### 大版本更新
仅当用户明确说"大版本更新"或"x.x大版本"时才执行大版本号变更（如 v3.x → v4.0），否则默认只加0.0.1。

## GitHub推送规则
- **仅在用户明确说"推送"时才推送到GitHub**
- 没有明确说"推送"时，所有修改只停留在本地工作区
- 不要主动推送，不要自动推送，不要猜测用户意图推送
- 每次推送前必须先问用户或等用户发命令
